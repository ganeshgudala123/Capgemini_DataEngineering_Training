# 🚀 Phase 2 – Translating SQL Concepts to PySpark (Joins & Aggregations)

## 📌 Overview
In this phase, I worked on implementing core SQL operations—specifically joins and aggregations—using PySpark. The tasks were based on practical, real-world datasets to strengthen applied understanding.

## 📚 Topics Covered
- Joins (Inner, Left)
- Aggregations (sum, avg, count)
- Filtering
- Sorting
- SQL to PySpark mapping


## ✅ Key Learnings
- Learned how to perform joins using `join()`
- Applied aggregations using `groupBy()`, `sum()`, `avg()`
- Understood how SQL queries translate into PySpark DataFrame operations
- Worked with real datasets (customers, orders)


## ⚠️ Challenges Faced
- Writing correct join conditions
- Understanding join types
- Handling missing values
- Applying multiple aggregations


## 📊 Tasks Implemented
1. Total order amount for each customer  
2. Top 3 customers by total spend  
3. Customers with no orders  
4. City-wise total revenue  
5. Average order amount per customer  
6. Customers with more than one order  
7. Sort customers by total spend  

---

## 🧠 Conclusion
This phase helped bridge the gap between SQL and PySpark by working on real-world scenarios involving joins and aggregations.
