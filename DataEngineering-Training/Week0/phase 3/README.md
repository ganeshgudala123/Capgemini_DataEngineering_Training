# 🚀 Phase 3 – ETL Pipeline using PySpark

## 📌 Overview
In this phase, we focused on building an end-to-end ETL (Extract, Transform, Load) pipeline using PySpark. The goal was to move from simple queries to real-world data engineering workflows.

---

## 📚 Topics Covered
- Data ingestion (CSV, JSON, Parquet)
- Data cleaning (handling null values)
- Filtering invalid records
- Joins and aggregations
- ETL pipeline design

---

## ✅ Key Learnings
- Understood ETL process (Extract → Transform → Load)
- Learned how to read and process datasets
- Handled missing values using `dropna()` and `fillna()`
- Applied transformations like filter, join, groupBy
- Built a pipeline for real-world use cases

---

## ⚠️ Challenges Faced
- Handling null values
- Converting incorrect data types
- Writing proper transformations
- Combining multiple steps into one pipeline

---

## 📊 Tasks Implemented
1. Read data and clean null values  
2. Filter invalid records  
3. Calculate daily sales  
4. Find repeat customers  
5. Find highest spending customer per city  
6. Build final reporting table  

---

## 🧠 Conclusion
This phase helped in understanding how real-world data pipelines are built using PySpark.
