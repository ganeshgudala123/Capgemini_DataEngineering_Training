# 🚀 Phase 3A – Data Quality & Cleaning Challenge

## 📌 Overview
In this phase, I worked with messy data and applied data cleaning techniques before performing any transformations. This helped in understanding the importance of data quality in data engineering.

---

## 📚 Topics Covered
- Handling null values
- Removing duplicates
- Filtering invalid data
- Data validation
- Basic aggregation

---

## ✅ Key Learnings
- Real-world data is often messy
- Cleaning data is essential before processing
- Invalid data leads to incorrect results
- Validation ensures data correctness

---

## ⚠️ Challenges Faced
- Handling missing values
- Identifying duplicate records
- Filtering incorrect data (negative age)
- Ensuring clean dataset before analysis

---

## 📊 Tasks Implemented
1. Identified data issues (nulls, duplicates, invalid values)  
2. Cleaned dataset using dropna() and filtering  
3. Removed duplicate records  
4. Validated data using row counts  
5. Performed aggregation (customers per city)  

---

## 🧠 Conclusion
This phase helped me understand that data cleaning is a critical step before building any data pipeline.
